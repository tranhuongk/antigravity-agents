---
description: Review Flutter/Dart changes for bugs, regressions, architecture issues, and missing tests.
---

When the user runs `/flutter-review <scope>`, use the `flutter-code-review` skill:

1. Inspect the diff or requested files.
2. Read surrounding code and similar patterns.
3. Do not edit files unless the user explicitly asks for fixes.
4. Lead with findings ordered by severity.
5. Include file and line references.
6. Include open questions, test gaps, and residual risk after findings.
