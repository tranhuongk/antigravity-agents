When the user runs `/codex-test <request>`, use the `codex-test-verification` skill:

1. Read existing test helpers and commands.
2. Add or update focused tests for success and failure paths when needed.
3. Run targeted tests first.
4. Run formatter, linter, type check, or broader tests when appropriate.
5. If coverage is requested, report the exact percentage.
6. Report what passed and what remains unverified.
