# Codex Docs Research

## Goal

Produce accurate, useful, source-grounded documentation or research output.

## Workflow

1. Read the project source of truth first:
   - README
   - docs
   - code comments
   - config files
   - tests
   - examples
2. For external or current facts, prefer primary sources:
   - official docs
   - specifications
   - source repositories
   - release notes
   - vendor docs
3. Distinguish fact, inference, and recommendation.
4. Update docs close to the behavior being described.
5. Keep examples executable or clearly marked as illustrative.
6. Verify commands and paths when feasible.

## Writing Standards

- Start with what the reader needs to do or know.
- Use concise headings and copy-pasteable commands.
- Avoid marketing language in technical docs.
- Mention prerequisites, assumptions, and failure modes.
- Keep docs consistent with existing tone and structure.

## Do Not

- Do not invent version-specific behavior.
- Do not cite secondary sources when official sources are available.
- Do not document future work as if it already exists.
- Do not include secrets, private URLs, or local machine paths.
